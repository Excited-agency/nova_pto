# Lessons Learned

## Правило: синхронізація тестів після змін

**Правило:** Після будь-якої зміни коду — автоматично перевіряти тести й оновлювати їх до того, як задача вважається виконаною.

**Чому:** Два gaps, виявлені після bugfix-сесії:
1. `admin-route.test.tsx` не мав тесту для `owner` ролі — хоча код вже приймав `owner`, тест це не перевіряв
2. `csv-validation.test.ts` не мав тесту для нового суворішого email-regex (`[a-zA-Z]{2,}` TLD) — граничний кейс `a@b.c` не був покритий

**Алгоритм:**
1. Для кожного зміненого файлу `X` — перевірити `src/test/**` на тести, що посилаються на `X`
2. Для кожного виправленого бага — запитати: "Чи є регресійний тест, що спіймає відкат цього виправлення?" Якщо ні — додати
3. Для змін у role/permission — завжди додавати тест для нової ролі
4. Для змін у валідації (regex, Zod schema) — додати тест на граничну умову
5. `npm test` наприкінці — кількість тестів має зрости на кількість нових кейсів
